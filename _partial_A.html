A-S1-004 POST /api/v1/auth/send-otp：发送短信验证码（含频率限制）
3hClaude Code
📄 文件
apps/api/src/routes/auth.ts apps/api/src/lib/rate-limiter.ts
🔨 步骤
  1. Zod 校验 body：phone 必须匹配 /^1[3-9]\d{9}$/，不合法返回 400 {code:'PHONE_INVALID'}
  2. Redis 频率检查：key = otp:rate:{phone}，1小时内 INCR 值超过5次返回 429 {code:'TOO_MANY_REQUESTS', retryAfter: X秒}
  3. 生成6位随机数字 OTP（crypto.randomInt(100000, 999999).toString()）
  4. bcrypt hash OTP（rounds=10），存入 OTP 表（expiresAt = now + 5min），同时将旧的未使用 OTP 标记为 used=true
  5. 调用 SMS API（dev 环境：console.log(\`[DEV OTP] ${phone}: ${otp}\`)，生产用阿里云短信或腾讯云）；返回 {success:true}
✅ 验收
dev 环境发送 OTP 后控制台打印验证码；同一手机号1小时内第6次请求返回 429；非法手机号返回 400

A-S4-004 apps/api/src/lib/cache.ts：Redis 缓存工具封装
2hCursor
📄 文件
apps/api/src/lib/cache.ts
🔨 步骤
  1. 封装 getOrSet<T>(key: string, ttlSeconds: number, fetchFn: () => Promise<T>): Promise<T>
  2. 逻辑：先 Redis GET key → 若有则 JSON.parse 返回 → 若无则执行 fetchFn → Redis SETEX key ttl JSON.stringify(result) → 返回 result
  3. 封装 invalidate(key: string | string[])：批量删除缓存 key（用于数据更新后）
  4. 在 GET /study/cards 和 GET /practice/questions 中使用，key 格式：vault:{id}:day:{n}:cards，TTL=600s
  5. 在 POST /study/complete 时调用 invalidate，使缓存失效
✅ 验收
第一次 GET /study/cards 耗时 ~200ms（DB查询），第二次 ~5ms（Redis缓存）；提交学习结果后缓存正确失效
