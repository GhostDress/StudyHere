📌 团队协作5条铁律

  1. 每个人完成一项任务后，必须在群里发通知（每个任务末尾有固定通知模板，直接复制发送）。
  2. 等前置任务通知，再开始自己的任务（任务里会写"等谁通知你才能开始"）。
  3. 遇到报错不要慌，把报错截图+Claude Code的回复原文 发到群里，让A来判断。
  4. 所有代码改动先提交到自己的分支，不要直接修改 main 分支（Claude Code会帮你做这件事）。
  5. 每天下班前把当天进度（做完了哪几项）发到群里，对齐进度。

A-001 在 GitHub 创建项目仓库 约30分钟
📖 做什么

⚠️ 开始前确认

  2. 点击右上角头像旁的「+」号，选「New repository」
  3. 仓库名（Repository name）填写：StudyHere

  7. 点击绿色的「Create repository」按钮
  8. 仓库创建成功后，页面地址形如 https://github.com/你的用户名/StudyHere —— 把这个地址复制保存好

A-003 配置 Supabase 云数据库 约1小时
📖 做什么

⚠️ 开始前确认
A-002 完成后进行。
🔨 具体步骤
  1. 打开浏览器，访问 supabase.com，点「Start your project」注册账号（可用 GitHub 账号一键登录）
  2. 进入控制台后，点「New project」
  3. 填写：项目名 StudyHere，数据库密码设一个强密码（必须记住！），区域选 Southeast Asia (Singapore)

  5. 项目创建完成后，点左侧菜单「Settings」→「API」
  6. 找到「Project URL」和「anon public」这两个值，各自点旁边的复制按钮，粘贴到记事本保存
  7. 再找到「service_role」的值，同样复制保存（这个要保密，不能公开）
  8. 把 Project URL 和 anon public key 私信发给 D
✅ 如何确认完成
